└ Masalah: binary yt-dlp tidak ada di folder tmp/
└ Akibat: semua command .ytmp3 .ytmp4 .play error
  saat mencoba menjalankan file yang tidak ada
└ Fix: download ulang binary yt-dlp v2026.03.17
  (Linux x86_64) ke tmp/yt-dlp dan chmod +x

> 📌 *YTMP3 / YTMP4 / Play — JavaScript Runtime Error*
└ Masalah: yt-dlp v2026 butuh JS runtime untuk
  ekstraksi YouTube, tapi tidak ada yang dikonfigurasi
└ Error: "No supported JavaScript runtime could be
  found. Only deno is enabled by default..."
└ Fix: tambah flag --js-runtimes node ke semua
  perintah yt-dlp (5 tempat: play download, ytmp3
  meta+download, ytmp4 meta+download)
└ Node.js v20 sudah tersedia di environment (/nix/...)
└ Tes berhasil: fetch meta YouTube return judul,
  durasi, channel dengan benar

> 📌 *YTMP3 / YTMP4 — Auto-Download yt-dlp*
└ Tambah fungsi ensureYtdlp() yang dipanggil sebelum
  setiap perintah download
└ Jika binary tidak ada, otomatis download dari
  GitHub releases (yt-dlp/yt-dlp) + chmod +x
└ Mencegah error berulang saat server restart atau
  binary terhapus oleh cleaner
